# AiToEarn 团队协作版改造实施计划

## 项目概述

**目标**: 将 AiToEarn 改造为支持 5 人团队协作的内部内容营销系统

**团队规模**: 5 人（权限相同，无需审核流程）

**支持平台**: 抖音、小红书、快手、B站、公众号

**内容类型**: 图文 + 短视频（<3分钟）

**服务器配置**: 阿里云轻量服务器（2核2G，60GB，Ubuntu）

**客户端环境**: Windows 11 + macOS 混合

---

## 技术架构

### 后端架构
```
Nginx (反向代理)
  ↓
NestJS Server (端口 7001)
  ↓
├── MongoDB (数据存储)
├── Redis (缓存 + 队列)
├── RustFS (文件存储)
└── 外部服务
    ├── MiniMax API (文案生成)
    ├── Doubao API (图片生成)
    └── 飞书 API (通知)
```

### 前端架构
```
Electron 客户端 (5个)
  ↓
├── React 19 + Next.js 15
├── Ant Design UI
├── Zustand 状态管理
└── WebSocket 实时通知
```

### 数据模型

#### 1. Team (团队)
```typescript
{
  _id: ObjectId,
  name: string,              // 团队名称
  description: string,       // 团队描述
  feishuWebhook: string,     // 飞书群机器人 Webhook
  createdAt: Date,
  updatedAt: Date
}
```

#### 2. Member (成员)
```typescript
{
  _id: ObjectId,
  teamId: ObjectId,
  userId: ObjectId,          // 关联 User
  name: string,
  email: string,
  avatar: string,
  status: 'active' | 'inactive',
  joinedAt: Date
}
```

#### 3. AccountGroup (账号分组)
```typescript
{
  _id: ObjectId,
  teamId: ObjectId,
  name: string,              // 分组名称（如"美食账号组"）
  platform: 'douyin' | 'xiaohongshu' | 'kuaishou' | 'bilibili' | 'wechat',
  accounts: [
    {
      accountId: ObjectId,   // 关联 Account
      nickname: string,
      avatar: string
    }
  ],
  createdAt: Date
}
```

#### 4. Content (内容)
```typescript
{
  _id: ObjectId,
  teamId: ObjectId,
  creatorId: ObjectId,       // 创作者
  type: 'image-text' | 'video',
  title: string,
  description: string,
  coverUrl: string,
  mediaUrls: string[],
  aiGenerated: {
    textByMiniMax: boolean,
    imageByDoubao: boolean
  },
  publishStatus: 'draft' | 'published',
  publishedTo: [
    {
      platform: string,
      accountId: ObjectId,
      publishedAt: Date,
      postId: string,
      url: string
    }
  ],
  createdAt: Date,
  updatedAt: Date
}
```

#### 5. Task (任务)
```typescript
{
  _id: ObjectId,
  teamId: ObjectId,
  title: string,
  description: string,
  assigneeId: ObjectId,      // 负责人
  creatorId: ObjectId,       // 创建人
  priority: 'low' | 'medium' | 'high',
  status: 'pending' | 'in-progress' | 'completed',
  dueDate: Date,
  relatedContentId: ObjectId,
  createdAt: Date,
  completedAt: Date
}
```

#### 6. Analytics (数据统计)
```typescript
{
  _id: ObjectId,
  teamId: ObjectId,
  accountId: ObjectId,
  platform: string,
  date: Date,                // 统计日期
  metrics: {
    publishCount: number,    // 发布量
    viewCount: number,       // 浏览量
    likeCount: number,       // 点赞数
    commentCount: number,    // 评论数
    shareCount: number,      // 分享数
    followerCount: number,   // 粉丝数
    followerGrowth: number   // 粉丝增长
  },
  createdAt: Date
}
```

## API 接口设计

### 1. 团队管理 API
```
GET    /api/team              - 获取团队信息
PUT    /api/team              - 更新团队信息
GET    /api/team/members      - 获取成员列表
POST   /api/team/members      - 添加成员
DELETE /api/team/members/:id  - 移除成员
```

### 2. 账号分组 API
```
GET    /api/account-groups           - 获取分组列表
POST   /api/account-groups           - 创建分组
PUT    /api/account-groups/:id       - 更新分组
DELETE /api/account-groups/:id       - 删除分组
POST   /api/account-groups/:id/accounts - 添加账号到分组
DELETE /api/account-groups/:id/accounts/:accountId - 从分组移除账号
```

### 3. 内容管理 API
```
GET    /api/contents          - 获取内容列表
POST   /api/contents          - 创建内容
PUT    /api/contents/:id      - 更新内容
DELETE /api/contents/:id      - 删除内容
POST   /api/contents/:id/publish - 发布内容
```

### 4. AI 生成 API
```
POST   /api/ai/generate-text  - MiniMax 文案生成
POST   /api/ai/generate-image - Doubao 图片生成
POST   /api/ai/reply-comment  - 智能评论回复
```

### 5. 任务管理 API
```
GET    /api/tasks             - 获取任务列表
POST   /api/tasks             - 创建任务
PUT    /api/tasks/:id         - 更新任务
DELETE /api/tasks/:id         - 删除任务
POST   /api/tasks/:id/complete - 完成任务
```

### 6. 数据统计 API
```
GET    /api/analytics/daily   - 日报数据
GET    /api/analytics/weekly  - 周报数据
GET    /api/analytics/monthly - 月报数据
GET    /api/analytics/compare - 账号对比数据
```

### 7. 通知 API
```
POST   /api/notifications/feishu - 发送飞书通知
GET    /api/notifications        - 获取通知列表
```

## Docker 部署配置（针对 2核2G 优化）

### docker-compose.yml
```yaml
version: '3.8'

services:
  mongodb:
    image: mongo:7.0
    container_name: aitoearn-mongodb
    restart: always
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_PASSWORD}
    command: --wiredTigerCacheSizeGB 0.5  # 限制内存使用

  redis:
    image: redis:7-alpine
    container_name: aitoearn-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru

  aitoearn-server:
    build:
      context: ./project/aitoearn-backend
      dockerfile: Dockerfile
    container_name: aitoearn-server
    restart: always
    ports:
      - "7001:7001"
    environment:
      NODE_ENV: production
      MONGODB_URI: mongodb://admin:${MONGO_PASSWORD}@mongodb:27017/aitoearn?authSource=admin
      REDIS_HOST: redis
      REDIS_PORT: 6379
      NODE_OPTIONS: --max-old-space-size=512  # 限制 Node.js 内存
    depends_on:
      - mongodb
      - redis

  aitoearn-ai:
    build:
      context: ./project/aitoearn-backend
      dockerfile: Dockerfile.ai
    container_name: aitoearn-ai
    restart: always
    ports:
      - "7002:7002"
    environment:
      NODE_ENV: production
      MINIMAX_API_KEY: ${MINIMAX_API_KEY}
      DOUBAO_API_KEY: ${DOUBAO_API_KEY}
      NODE_OPTIONS: --max-old-space-size=512
    depends_on:
      - aitoearn-server

  nginx:
    image: nginx:alpine
    container_name: aitoearn-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - aitoearn-server

volumes:
  mongodb_data:
  redis_data:
```

### 内存使用预估
- MongoDB: ~500MB
- Redis: ~256MB
- aitoearn-server: ~512MB
- aitoearn-ai: ~512MB
- Nginx: ~50MB
- 系统开销: ~200MB
- **总计**: ~2GB（刚好符合服务器配置）

## 风险与应对

### 1. 服务器性能风险
**风险**: 2核2G 配置可能在高并发时性能不足

**应对**:
- 实施严格的内存限制
- 使用 Redis 缓存减少数据库查询
- 异步处理耗时任务（AI 生成、视频处理）
- 监控服务器资源使用，及时优化

### 2. AI 服务稳定性风险
**风险**: MiniMax/Doubao API 可能不稳定或限流

**应对**:
- 实施重试机制
- 添加降级方案（手动输入）
- 监控 API 调用成功率
- 准备备用 AI 服务

### 3. 平台 API 变更风险
**风险**: 各平台 API 可能变更导致发布失败

**应对**:
- 模块化设计，便于快速适配
- 监控发布成功率
- 保留手动发布入口
- 及时更新 SDK

### 4. 数据安全风险
**风险**: 账号信息、内容数据泄露

**应对**:
- 敏感信息加密存储
- 实施访问控制
- 定期数据备份
- 服务器安全加固
