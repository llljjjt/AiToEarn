## 环境准备

### 服务器要求
- **操作系统**: Ubuntu 20.04+ / CentOS 7+
- **配置**: 2核2G（最低）/ 4核8G（推荐）
- **磁盘**: 40GB+
- **网络**: 公网 IP，开放端口 80, 443, 3000, 27017, 6379

### 必需软件
```bash
# 1. 安装 Docker
curl -fsSL https://get.docker.com | bash -s docker
sudo systemctl start docker
sudo systemctl enable docker

# 2. 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 3. 安装 Node.js 18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 4. 安装 pnpm
npm install -g pnpm
```

## 后端部署

### 1. 克隆项目
```bash
cd /opt

cd aitoearn
```

### 2. 配置环境变量
```bash
# 复制环境变量模板
cp project/aitoearn-electron/server/.env.example .env

# 编辑环境变量
nano .env
```

**关键配置项**:
```bash
# 服务器配置
NODE_ENV=production
SERVER_PORT=3000

# MongoDB 配置
MONGO_URI=mongodb://root:YOUR_PASSWORD@localhost:27017/?directConnection=true
MONGO_DB=aitoearn_team

# Redis 配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=YOUR_REDIS_PASSWORD
REDIS_DB=0

# 阿里云 OSS（文件存储）
OSS_KEY_ID=your_oss_key_id
OSS_KEY_SECRET=your_oss_key_secret
OSS_REGION=oss-cn-hangzhou
OSS_BUCKET=your_bucket_name
OSS_URL=https://your_bucket.oss-cn-hangzhou.aliyuncs.com

# 阿里云 SMS（短信验证）
SMS_ACCESS_KEY_ID=your_sms_key_id
SMS_ACCESS_KEY_SECRET=your_sms_key_secret
SMS_SIGN_NAME=your_sign_name
SMS_TEMPLATE_CODE=your_template_code

# AI 服务（可选）
QWEN_KEY=your_qwen_key

# 飞书通知（团队协作必需）
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_WEBHOOK_TOKEN
```

### 3. 优化 Docker Compose 配置（2核2G 服务器）

创建优化后的 `docker-compose.prod.yml`:

```yaml
version: '3.8'

services:
  # MongoDB 副本集
  mongo1:
    image: mongo:7.0
    container_name: aitoearn-mongo1
    restart: always
    command: mongod --replSet rs0 --bind_ip_all --wiredTigerCacheSizeGB 0.5
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: ${MONGO_PASSWORD:-your_secure_password}
    volumes:
      - ./data/mongo1:/data/db
    ports:
      - "27017:27017"
    networks:
      - aitoearn-network
    mem_limit: 512m
    cpus: 0.5

  # Redis
  redis:
    image: redis:7-alpine
    container_name: aitoearn-redis
    restart: always
    command: redis-server --requirepass ${REDIS_PASSWORD:-your_redis_password} --maxmemory 256mb --maxmemory-policy allkeys-lru
    volumes:
      - ./data/redis:/data
    ports:
      - "6379:6379"
    networks:
      - aitoearn-network
    mem_limit: 256m
    cpus: 0.3

  # 后端服务
  backend:
    build:
      context: ./project/aitoearn-backend
      dockerfile: Dockerfile
    container_name: aitoearn-backend
    restart: always
    env_file:
      - .env
    environment:
      - NODE_ENV=production
      - SERVER_PORT=3000
    ports:
      - "3000:3000"
    depends_on:
      - mongo1
      - redis
    networks:
      - aitoearn-network
    mem_limit: 768m
    cpus: 1.0
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: aitoearn-nginx
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      - backend
    networks:
      - aitoearn-network
    mem_limit: 128m
    cpus: 0.2

networks:
  aitoearn-network:
    driver: bridge

volumes:
  mongo1-data:
  redis-data:
```

### 4. 创建 Nginx 配置

创建 `nginx/nginx.conf`:

```nginx
user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;
    client_max_body_size 100M;

    gzip on;
    gzip_vary on;
    gzip_proxied any;
    gzip_comp_level 6;
    gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss application/rss+xml font/truetype font/opentype application/vnd.ms-fontobject image/svg+xml;

    upstream backend {
        server backend:3000;
    }

    server {
        listen 80;
        server_name _;

        location / {
            proxy_pass http://backend;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            proxy_cache_bypass $http_upgrade;
            proxy_read_timeout 300s;
            proxy_connect_timeout 75s;
        }

        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }
    }
}
```

### 5. 构建和启动服务

```bash
# 1. 安装后端依赖
cd project/aitoearn-backend
pnpm install

# 2. 构建后端
pnpm build

# 3. 返回项目根目录
cd ../..

# 4. 启动 Docker 服务
docker-compose -f docker-compose.prod.yml up -d

# 5. 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 6. 查看日志
docker-compose -f docker-compose.prod.yml logs -f backend
```

### 6. 初始化 MongoDB 副本集

```bash
# 进入 MongoDB 容器
docker exec -it aitoearn-mongo1 mongosh -u root -p your_secure_password

# 初始化副本集
rs.initiate({
  _id: "rs0",
  members: [
    { _id: 0, host: "mongo1:27017" }
  ]
})

# 检查副本集状态
rs.status()

# 退出
exit
```



## 常见问题

### 1. MongoDB 连接失败

**问题**: `MongoServerError: Authentication failed`

**解决**:
```bash
# 1. 检查 MongoDB 是否启动
docker ps | grep mongo

# 2. 检查密码是否正确
docker exec -it aitoearn-mongo1 mongosh -u root -p your_password

# 3. 重置密码
docker exec -it aitoearn-mongo1 mongosh
use admin
db.changeUserPassword("root", "new_password")
```

### 2. Redis 连接失败

**问题**: `Error: Redis connection to localhost:6379 failed`

**解决**:
```bash
# 1. 检查 Redis 是否启动
docker ps | grep redis

# 2. 测试连接
docker exec -it aitoearn-redis redis-cli -a your_redis_password
PING
# 应返回 PONG

# 3. 检查密码
docker logs aitoearn-redis
```

### 3. 后端服务启动失败

**问题**: `Error: Cannot find module '@nestjs/core'`

**解决**:
```bash
# 1. 重新安装依赖
cd project/aitoearn-backend
rm -rf node_modules pnpm-lock.yaml
pnpm install

# 2. 重新构建
pnpm build

# 3. 重启服务
docker-compose -f docker-compose.prod.yml restart backend
```

### 4. Electron 客户端无法连接服务器

**问题**: `Network Error: connect ECONNREFUSED`

**解决**:
1. 检查服务器 IP 是否正确
2. 检查服务器防火墙是否开放 3000 端口
3. 检查服务器是否启动

```bash
# 开放防火墙端口（Ubuntu）
sudo ufw allow 3000/tcp

# 开放防火墙端口（CentOS）
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --reload

# 测试连接
curl http://your-server-ip:3000/health
```

### 5. 内存不足

**问题**: `FATAL ERROR: Reached heap limit Allocation failed`

**解决**:
```bash
# 1. 增加 Node.js 内存限制
export NODE_OPTIONS="--max-old-space-size=512"

# 2. 优化 MongoDB 缓存
# 编辑 docker-compose.prod.yml
command: mongod --replSet rs0 --bind_ip_all --wiredTigerCacheSizeGB 0.3

# 3. 优化 Redis 内存
command: redis-server --maxmemory 128mb --maxmemory-policy allkeys-lru

# 4. 重启服务
docker-compose -f docker-compose.prod.yml restart
```

### 6. 飞书通知发送失败

**问题**: `Error: Feishu webhook request failed`

**解决**:
1. 检查 Webhook URL 是否正确
2. 检查飞书机器人是否被禁用
3. 检查消息格式是否正确

```bash
# 测试 Webhook
curl -X POST "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "msg_type": "text",
    "content": {
      "text": "测试消息"
    }
  }'
```

## 安全建议

1. **修改默认密码**: MongoDB、Redis 使用强密码
2. **启用 HTTPS**: 使用 Let's Encrypt 免费证书
3. **配置防火墙**: 只开放必要端口（80, 443）
4. **定期备份**: 每天自动备份数据库
5. **日志审计**: 定期检查访问日志
6. **更新依赖**: 定期更新 Node.js 和依赖包
