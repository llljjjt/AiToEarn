## 需要的操作

### 1. 通过阿里云控制台恢复服务器

登录阿里云控制台 → 找到服务器 47.236.74.56 → 使用 VNC 远程连接

检查服务器状态：
```bash
# 查看资源使用
top
free -h
df -h

# 检查 SSH 服务
systemctl status sshd

# 检查是否有卡住的进程
ps aux | grep nx
ps aux | grep node
ps aux | grep pnpm
```

恢复方案：
```bash
# 方案 1: 重启 SSH 服务
systemctl restart sshd

# 方案 2: 杀死卡住的进程
pkill -9 nx
pkill -9 node

# 方案 3: 重启服务器（如果资源耗尽）
reboot
```

### 2. 服务器恢复后执行部署

在本地执行部署脚本：
```bash
cd /e/小红书分发/AiToEarn-main
bash deploy-to-server.sh
```

或者手动执行：
```bash
# 1. 上传文件
scp /tmp/backend-dist.tar.gz root@47.236.74.56:/opt/aitoearn/
scp /tmp/frontend-dist.tar.gz root@47.236.74.56:/opt/aitoearn/
scp docker-compose.prod.yml root@47.236.74.56:/opt/aitoearn/

# 2. 连接服务器
ssh root@47.236.74.56

# 3. 解压和部署
cd /opt/aitoearn
tar -xzf backend-dist.tar.gz
tar -xzf frontend-dist.tar.gz
cd dist/apps/aitoearn-server
npm install --production
cd /opt/aitoearn

# 4. 启动服务
docker-compose -f docker-compose.prod.yml up -d

# 5. 检查状态
docker-compose -f docker-compose.prod.yml ps
docker-compose -f docker-compose.prod.yml logs -f
```

### 3. 验证部署

```bash
# 检查服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs backend

# 测试 API
curl http://47.236.74.56:3000/health
curl http://47.236.74.56:3000/api/team
```

## 优化建议

### 避免服务器资源耗尽
1. **不要在 2核2G 服务器上构建**：
   - TypeScript 编译非常消耗资源
   - 应该在本地或 CI 服务器构建，只部署编译产物

2. **使用 CI/CD 流程**：
   - GitHub Actions / GitLab CI
   - 在云端构建，自动部署到服务器

3. **监控服务器资源**：
   ```bash
   # 安装监控工具
   apt install htop iotop
   
   # 设置资源限制
   ulimit -m 1500000  # 限制内存使用
   ```

### Docker Compose 优化
已在 `docker-compose.prod.yml` 中配置：
- MongoDB: 限制 1GB 内存，缓存 512MB
- Redis: 限制 128MB 内存
- Backend: 限制 512MB 内存
